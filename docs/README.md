# Documentation Assets

This directory contains screenshots and demo images for the Sentinel Architect project.

## Image Files

Place your screenshot files in this directory with the following naming convention:

- `demo-main-interface.png` - Main application interface
- `demo-config-panel.png` - Configuration panel view
- `demo-spec-generation.png` - Specification generation output
- `demo-code-generation.png` - Code generation output
- `demo-processing.png` - Processing/loading state
- `demo-secure-kiosk.png` - Secure kiosk environment
- `demo-network-blocked.png` - Network blocking example
- `demo-network-allowed.png` - Network allowed example
- `demo-threat-detection-single.png` - Single threat detection
- `demo-threat-detection-multiple.png` - Multiple threats detected
- `demo-security-lockdown.png` - Security lockdown state
- `demo-threat-chrome.png` - Chrome browser threat
- `demo-threat-cheat-engine.png` - Cheat Engine threat
- `demo-threat-discord.png` - Discord threat
- `demo-threat-registry.png` - Registry process anomaly
- `demo-kernel-logs.png` - Kernel event logs
- `demo-chaos-monkey.png` - Chaos Monkey panel

## Image Specifications

- **Format**: PNG (recommended) or JPG
- **Width**: 800px (for main screenshots)
- **Width**: 400-600px (for smaller panels)
- **Aspect Ratio**: Maintain original aspect ratio
- **File Size**: Optimize images to keep repository size reasonable (< 500KB per image)

## Adding Images

1. Take screenshots of your application in various states
2. Name them according to the convention above
3. Place them in this `docs/` directory
4. The README.md will automatically display them

## Alternative: Using GitHub Assets

If you prefer to host images elsewhere:
1. Upload images to GitHub Issues or use GitHub's asset hosting
2. Update the image paths in the main README.md to use full URLs
3. Example: `https://github.com/user-attachments/assets/...`
